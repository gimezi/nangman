# 프로젝트 개요
마비노기 모바일 길드 사이트
- 사용자는 두가지 타입으로 나뉨
  - 길드원 / 임원
  - 길드원은 자신의 캐릭터를 등록 / 삭제 / 수정 / 조회 할 수 있음
  - 길드원은 자신의 캐릭터들을 원하는 레이드에 지원할 수 있음
  - 임원은 다른 길드원들의 캐릭터들을 전체 조회할 수 있음
  - 임원은 레이드를 등록 / 삭제 / 수정 / 조회할 수 있음
  - 임원은 레이드마다 등록한 캐릭터들을 조합해서 레이드마다 파티를 n개 만들 수 있음
- 레이드마다 필요한 파티원의 수는 다름
- 레이드마다 지원 가능한 시간이 정해져있음
- 해당 시간이 지난 후, 전투력과 직업을 고려해서 파티를 자동으로 배치시킴
  - 이때, 배치된 파티는 임원이 바꿀 수 있음
  - 파티는 보통 같은 시간에 2개씩 운영됨
  - 한 사람이 동시에 진행하는 파티에 들어갈 수 없음
  - 빈 자리가 있을 수 있음(빈자리가 있을 경우, 관리자가 임의로 다른 파티의 유저를 중복 참여 시킬 수 있음, 단 다른 시간에 한해서)
  - 직업의 목록은 src/models/classes.ts에 저장해두었음
  - 각 파티는 평균 전투력이 비슷하게 짜여져야 함
  - 직업의 밸런스는 다음과 같음
    - 매우 강한 딜러가 있다면 support나 tank가 적거나 없어도 가능
    - 파티마다 최소 support가 한명은 있어야 함
    - 파티마다 tank는 비슷하게 들어가야함


# 기술 스택 및 아키텍처
- 일단 초기 데이터는 구글 스프레드 시트의 길드원의 전투력과 직업 리스트를 가져와서 세팅
- 프론트엔드와 백엔드는 Next.js 사용, 데이터베이스는 supabase 사용, 배포는 vercel 사용
- 로그인은 단순히 자신의 닉네임 + 비밀번호

# 구현할 페이지
1. 로그인 페이지
    - 자신의 닉네임과 비밀번호 입력(한 번만 로그인해도 이후엔 자동로그인)
2. 회원가입 페이지
    - 자신의 닉네임 입력 후 확인 -> 데이터 베이스 길드원 목록에 있는지 확인 있으면 200
    - 비밀번호 (숫자 4자리만 입력), 비밀번호 확인
3. 캐릭터 목록 페이지
   - 스프레드 시트에서 가져온 본인의 닉네임에 해당되는 캐릭터 목록 가져옴
   - 캐릭터별 닉네임, 직업, 전투력이 나옴
   - 수정 및 삭제 가능
   - 새로운 캐릭터 모달로 등록 가능
4. 레이드 목록 페이지
    - 상단 탭에서 레이드를 선택(일단은 타매어만)
    - 레이드는 일단 두가지만 존재(매주 월 / 화)
    - 각 레이드를 선택하면 레이드 참여 캐릭터 선택화면으로 넘어감
5. 레이드 참여 캐릭터 선택
    - 참여를 하려는 레이드 정보
    - 내 캐릭터 목록이 나옴
    - 내 캐릭터를 다중선택해서 등록하기
6. (관리자) 모든 사람들의 캐릭터 목록 페이지
    - 모든 유저의 캐릭터 목록을 확인할 수 있음
    - 대표 유저 닉네임 / 닉네임 / 직업 / 전투력 이 나옴
    - 수정 및 삭제, 등록이 가능함
    - 여기서 대표 유저를 등록하면 새로운 닉네임을 가진 길드원이 회원가입할 수 있음
    - 유저를 선택해서 관리자로 만들거나, 일반유저로 전환시킬 수 있음
7. (관리자) 레이드 관리 페이지
   - 레이드 (대분류) 등록 및 삭제
     - 레이드 소분류를 같이 등록함. 매주 요일과 레이드 이름, 대표 사진, 권장 전투력, 최소 전투력, 파티 당 인원을 설정 가능
   - 각 레이드 별 참여 신청 인원을 볼 수 있음
   - 각 레이드 별 신청 기간을 정할 수 있음(ex. 매주 월요일 파티면 월요일 n시전까지)
8. (관리자) 레이드 파티 관리 페이지
   - 위의 파티 조건을 참고해서 참여 신청한 인원을 배치 시킴
   - 파티는 한번에 1~3개 진행 가능, 기본은 2개(해당 페이지에서 숫자 조절가능, 숫자 조절 시 다시 배치)
   - 각각의 유저를 드래그, 드랍해서 구성을 변경할 수 있음
   - 파티마다 직업과 투력이 한눈에 보이고 평균 투력도 확인 가능


# 구현 완료 기능

## Discord 웹훅 연동 (src/app/api/admin/discord/)
- `route.ts`: 어드민 전용 API. `next/og`의 `ImageResponse`로 팀별 파티 구성 이미지(PNG) 생성 후 Discord 포럼 채널에 파일 첨부로 전송
  - 폰트: jsDelivr에서 NotoSansKR woff(woff2 아님, Satori 미지원)를 런타임 로드 후 메모리 캐시
  - `partyImage.tsx`: 팀별 JSX 레이아웃 (팀 색상, 파티 카드, 멤버/직업/전투력)
  - Discord 포럼 채널은 반드시 `thread_name` 필요, `?wait=true` 붙여야 응답 JSON 받음
- `/parties` 페이지에 어드민 전용 "파티 수정" 버튼 + "디스코드 공유" 버튼 표시

## /parties 페이지 (src/app/(main)/parties/)
- 저장된 파티 중 가장 최근 주차를 자동으로 표시 (다음 주 아님)
- 팀별 그리드 레이아웃 (홍팀/청팀 나란히)
- 파티 박스 클릭 시 "홍팀 1파티 - [닉네임/닉네임/...]" 형식으로 클립보드 복사 + 스낵바
- 어드민 전용 버튼: "파티 수정" (→ /admin/parties?scheduleId=&weekDate=), "디스코드 공유"

## /admin/parties 수정 진입 (src/app/(main)/admin/parties/)
- URL 파라미터 `scheduleId`, `weekDate`로 진입 시 해당 스케줄/주차 자동 선택
- 저장된 파티가 있으면 팀 수 버튼 없이도 자동 로드 (numTeams 체크 제거)
- 첫 마운트 시 스케줄 reset effect 건너뜀 (isFirstScheduleEffect ref)

## 구글 시트 동기화 (src/app/api/admin/sync-sheet/)
- 공개 구글 스프레드시트 CSV를 파싱해서 레이드 신청 자동 등록
- 한국어 타임스탬프("2026. 5. 11 오후 3:30:52") 파싱 후 가장 가까운 월요일 날짜로 매핑
- 환경변수: `GOOGLE_SHEET_ID`, `GOOGLE_SHEET_GID`

## 환경변수 (.env.local)
- `DISCORD_WEBHOOK_URL`: Discord 포럼 채널 웹훅 URL
- `GOOGLE_SHEET_ID`, `GOOGLE_SHEET_GID`: 구글 시트 연동
- `JWT_SECRET`, `NEXT_PUBLIC_SUPABASE_*`, `SUPABASE_SERVICE_ROLE_KEY`: 기존
